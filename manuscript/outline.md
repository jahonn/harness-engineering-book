# 《Harness Engineering》全书目录

## 第一编 为什么需要 Harness Engineering

### 第一章 为什么需要 Harness Engineering

目标：定义问题域，解释为什么更强模型并不能自动带来可靠智能体系统，建立“模型能力需要运行基底”的主论点。

主要来源：HE-001、HE-002、HE-003、匿名架构复盘案例、匿名内部成熟度复盘。

### 第二章 模型、Harness 与环境的系统边界

目标：给出模型、harness、环境、用户、外部系统之间的边界，建立后续全书的系统图。

主要来源：HE-001、HE-003、HE-007、匿名架构复盘案例。

### 第三章 从 Prompt Engineering 到 Harness Engineering

目标：说明提示词工程的价值与边界，解释为什么生产级智能体必须进入上下文、工具、状态、权限和评测工程。

主要来源：HE-003、HE-004、HE-008、HE-009。

### 第四章 智能体系统的失败模式

目标：系统归纳幻觉、错误工具选择、权限越界、状态漂移、上下文污染、不可回滚、不可观测和评测失真。

主要来源：HE-002、HE-005、HE-006、匿名内部成熟度复盘。

## 第二编 Harness 的核心结构

### 第五章 模型契约与能力边界

讨论模型选择、上下文窗口、输出预算、工具调用能力、多模态能力、推理内容、供应商特定行为和抽象边界。

### 第六章 上下文装配

讨论系统指令、用户目标、项目规则、检索材料、历史压缩、任务摘要、优先级和冲突处理。

### 第七章 Agent Loop

讨论思考、行动、观察、更新和停止条件，比较一次性调用、多轮工具调用、规划执行和反思循环。

### 第八章 工具系统

讨论工具 schema、参数验证、输出裁剪、错误语义、幂等性、并发、超时、重试和工具选择。

### 第九章 文件系统与工作区

讨论 coding agent 的工作区边界、读写策略、搜索、patch、diff、临时文件、生成物和路径安全。

### 第十章 记忆与长期上下文

讨论用户偏好、项目规则、长期事实、经验沉淀、作用域、冲突、过期和可解释更新。

## 第三编 安全、权限与治理

### 第十一章 权限模型

讨论工具级、风险级、路径级、模式级、用户级和组织级权限。

### 第十二章 Sandbox 与隔离

讨论本地沙箱、容器、网络、凭据、环境变量、系统命令和危险操作。

### 第十三章 人工审批与交互设计

讨论什么时候必须 ask、如何展示风险、如何避免审批疲劳、如何保留审计记录。

### 第十四章 Guardrail

讨论输入、输出、工具调用、外部系统动作和政策判断的护栏。

### 第十五章 回滚、恢复与事故处理

讨论 checkpoint（检查点）、git、快照、撤销、失败恢复、事故复盘和数据一致性。

## 第四编 可观测性与评测

### 第十六章 Trace 与日志

讨论工具调用 trace（轨迹）、token、成本、错误、审批、状态变化和隐私脱敏。

### 第十七章 评测对象从模型到系统

讨论为什么智能体评测必须覆盖环境、工具、状态和最终产物。

### 第十八章 软件工程智能体评测

讨论 SWE-bench、Terminal-Bench、仓库任务、测试通过率、补丁质量和人工审稿。

### 第十九章 成本、延迟与容量

讨论 token 预算、工具调用成本、重试、长上下文、并发和任务队列。

### 第二十章 质量门禁

讨论自动测试、类型检查、lint、静态分析、人工验收和发布前审查。

## 第五编 产品化 Agent OS

### 第二十一章 从 Harness Core 到 Agent OS

讨论会话、命令、插件、profile、任务队列、远程运行和 UI。

### 第二十二章 终端式 Coding Agent

讨论 TUI、命令面板、diff 视图、审批流、timeline、composer 和 inspector。

### 第二十三章 多智能体调度

讨论 subagent、DAG、fan-out、fan-in、角色、模型覆盖、失败传播和并发上限。

### 第二十四章 插件与协议

讨论 MCP、工具市场、插件 manifest、信任模型、权限继承和版本兼容。

### 第二十五章 企业集成

讨论身份、审计、代码仓库、issue、CI、文档、知识库、消息系统和审批系统。

## 第六编 Harness 的持续演化

### 第二十六章 观测驱动的演化

讨论从 trace（轨迹）、失败样本、人工反馈和评测结果推动 harness 改进。

### 第二十七章 自动化 Harness 改进

讨论 agentic harness engineering、提示、工具、流程和评测闭环的自动优化。

### 第二十八章 经验沉淀为规则、技能与测试

讨论如何把一次失败变成项目规则、系统 prompt、技能、工具校验或测试。

### 第二十九章 版本管理与迁移

讨论 prompt、工具 schema、profile、记忆、eval 和插件的版本管理。

### 第三十章 组织学习

讨论团队如何运行 agent 平台治理、审稿、培训、事故复盘和可复用做法传播。
